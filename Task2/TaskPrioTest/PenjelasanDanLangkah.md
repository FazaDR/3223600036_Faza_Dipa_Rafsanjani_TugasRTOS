Pada percobaan ini, program dibuat untuk menguji mekanisme prioritas task pada sistem operasi real-time (FreeRTOS). Dua buah task dijalankan secara bersamaan dan keduanya dikunci pada core 1 menggunakan fungsi xTaskCreatePinnedToCore(). Task pertama (task1) memiliki prioritas lebih rendah (1), sedangkan task kedua (task2) memiliki prioritas lebih tinggi (2). Kedua task berfungsi untuk menyalakan dan mematikan LED pada pin yang berbeda (LED1 di pin 6 dan LED2 di pin 1). Namun, task1 dan task2 tidak menggunakan vTaskDelay(), sehingga keduanya akan berusaha terus berjalan tanpa memberi waktu CPU untuk task lain.

Hasil percobaan menunjukkan bahwa hanya LED pada task dengan prioritas lebih tinggi (task2) yang akan tampak aktif, sementara LED1 tidak akan menyala sama sekali. Juga bisa dilihat serial tercetak hanyalah pada task2. Hal ini terjadi karena task2 memiliki prioritas lebih tinggi dan tidak pernah menyerahkan kendali CPU (tidak ada vTaskDelay() atau taskYIELD()), sehingga task1 tidak pernah mendapat giliran dieksekusi. Percobaan ini memperlihatkan prinsip utama FreeRTOS, yaitu task dengan prioritas lebih tinggi akan selalu dijalankan terlebih dahulu, dan task dengan prioritas lebih rendah hanya berjalan jika task yang lebih tinggi sedang dalam keadaan “blocked” atau “yield”.
